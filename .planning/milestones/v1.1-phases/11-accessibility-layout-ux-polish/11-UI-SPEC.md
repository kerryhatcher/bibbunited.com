## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`gap-4`, `p-4`, `pt-16` header offset) |
| lg | 24px | Section padding (`gap-6`, `p-6`) |
| xl | 32px | Layout gaps (`--spacing-section-mobile`) |
| 2xl | 48px | Major section breaks (`py-12`, `mb-12`) |
| 3xl | 64px | Page-level spacing (`--spacing-section`) |

Exceptions:
- `pt-16` (64px) on `<main>` is NOT decorative -- it compensates for the sticky header (`h-16` = 64px). Do not remove. (D-15, D-16)
- Touch targets: all interactive elements use `min-h-[44px]` per existing Button component pattern.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (`--color-bg-dominant`) | Page background, card backgrounds |
| Secondary (30%) | `#1B2A4A` (`--color-navy` / `--color-bg-secondary`) | Header border, footer background, dark sections |
| Accent (10%) | `#DC2626` (`--color-accent` / `--color-crimson`) | See reserved list below |
| Destructive | same as accent (`#DC2626`) | No separate destructive actions in this phase |

Accent reserved for:
- Active nav bottom border (desktop) -- 3px solid accent
- Active nav left bar (mobile) -- 3px solid accent
- Footer focus ring color on dark background -- `focus-visible:ring-accent`
- CTA buttons (existing pattern)
- Link hover states (existing pattern)

**Focus indicator on dark background (footer):** Use `--color-accent` (#DC2626 crimson) for `focus-visible:ring-2`. This provides 5.9:1 contrast ratio against navy (#1B2A4A), exceeding WCAG 2.1 AA 3:1 minimum for non-text elements. Ring offset uses transparent or navy to prevent white gap.

**Active nav indicator colors:** Use `--color-accent` (#DC2626) for both desktop bottom border and mobile left bar. Consistent with the site's bold, urgent brand identity.

## Interaction Contracts

### Skip Link Behavior
1. User loads any page. Skip link is off-screen (`sr-only` positioning).
2. User presses Tab. Skip link slides into view at top-left of viewport.
3. User presses Enter. Focus moves to `<main id="main-content">`. Page scrolls to main content.
4. User presses Tab again (without Enter). Skip link slides back off-screen. Focus continues to next focusable element (header logo).

### Active Nav Detection
1. `usePathname()` returns current route (e.g., `/news`).
2. Compare against each nav item's resolved href via `resolveHref()`.
3. Exact match: set `aria-current="page"` and apply active visual styles.
4. No partial matching -- `/news/some-slug` does NOT activate the "News" nav item. Only exact path match.

### Footer CTA Conditional Rendering
1. Read current pathname (requires client component or client wrapper).
2. For each CTA button, compare `href` against current pathname.
3. If match, do not render that button.
4. If both buttons are hidden, hide the entire CTA section (heading + paragraph + button container).

### Excerpt Display on News Cards
1. Query includes `excerpt` field from NewsPost collection.
2. If `post.excerpt` exists and is non-empty, display it.
3. If `post.excerpt` is empty, extract ~150 characters of plain text from `post.body` (Lexical rich text), truncate at word boundary, append ellipsis.
4. Display excerpt in `<p>` tag with `line-clamp-2` (max 2 visible lines, CSS overflow hidden).

## Accessibility Contracts

| Requirement | Implementation | WCAG Criterion |
|-------------|---------------|----------------|
| A11Y-01: Homepage H1 | `<h1 class="sr-only">` as first child of homepage content | 1.3.1 Info and Relationships |
| A11Y-02: Skip link | `<a href="#main-content">` with sr-only + focus:visible pattern | 2.4.1 Bypass Blocks |
| A11Y-04: Footer focus rings | `focus-visible:ring-2 focus-visible:ring-accent` on all footer interactives | 2.4.7 Focus Visible |
| UX-02: Active nav | `aria-current="page"` on active link | 1.3.1 Info and Relationships |

Focus ring specification:
- Width: 2px (`ring-2`)
- Color: `--color-accent` (#DC2626)
- Offset: 2px (`ring-offset-2`)
- Offset color on dark backgrounds: `ring-offset-navy` (#1B2A4A)
- Trigger: `:focus-visible` only (not `:focus`) -- keyboard users see rings, mouse users do not.
