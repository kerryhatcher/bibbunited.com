    **## Cloudflare DNS Configuration**
    Document the following steps:
    1. Create a CNAME record pointing `www.bibbunited.com` to the Cloudflare tunnel hostname (format: `<tunnel-id>.cfargotunnel.com`)
    2. Create a CNAME record pointing `bibbunited.com` (root) to `www.bibbunited.com` or use Cloudflare's CNAME flattening
    3. In Cloudflare Page Rules (or Cache Rules):
       - Create rule: URL pattern `*bibbunited.com/admin*` with Cache Level = Bypass
       - This ensures the Payload CMS admin panel is never served from cache
    4. SSL/TLS mode should be set to "Full (strict)" since the tunnel provides end-to-end encryption

    **## Docker Build Requirements**
    Document that:
    1. `next build` (run during Docker image build stage) requires a live PostgreSQL connection because Payload CMS initializes its database schema during build
    2. The `DATABASE_URI` environment variable must be available at build time
    3. For CI/CD (GitHub Actions), use a PostgreSQL service container:
       ```yaml
       services:
         postgres:
           image: postgres:16-alpine
           env:
             POSTGRES_USER: payload
             POSTGRES_PASSWORD: payload
             POSTGRES_DB: payload_build
           ports:
             - 5432:5432
           options: >-
             --health-cmd pg_isready
             --health-interval 10s
             --health-timeout 5s
             --health-retries 5
       ```
    4. Set build arg: `DATABASE_URI=postgresql://payload:payload@localhost:5432/payload_build`
    5. This is a build-only database -- it does not need to persist data

    **## JSON-LD Structured Data Validation**
    Document that:
    1. JSON-LD structured data (Organization, WebSite, NewsArticle schemas) is rendered on relevant pages
    2. Validation requires a publicly accessible URL -- cannot be tested against localhost
    3. After deployment, validate using Google's Rich Results Test: https://search.google.com/test/rich-results
    4. Enter the live site URL (e.g., `https://www.bibbunited.com/news/article-slug`)
    5. Verify that Organization and Article schemas are detected without errors
    6. Also test with Schema.org validator: https://validator.schema.org/

    **## Media Upload Persistence**
    Document that:
    1. Media files are stored at `/app/media/` inside the container
    2. K8s deployment mounts a PersistentVolumeClaim at this path
    3. The PVC must use a storage class that supports ReadWriteOnce (or ReadWriteMany if running multiple replicas)
    4. Verify persistence: upload an image via admin, restart the pod, confirm the image is still accessible

    **## Environment Variables**

    - `DATABASE_URI` -- PostgreSQL connection string (required at build time AND runtime)
    - `PAYLOAD_SECRET` -- Random string for Payload session encryption (min 32 chars)
    - `NEXT_PUBLIC_SERVER_URL` -- Public URL of the site (e.g., `https://www.bibbunited.com`)
