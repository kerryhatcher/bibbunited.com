## Component Inventory

Only one new component surface is introduced. All others are existing components reused as-is.

### New: `/news` listing page (`src/app/(frontend)/news/page.tsx`)

**Layout:** Server component. Section wrapper (default variant) containing:
1. Page title: `<h1>` with display typography, uppercase, Barlow Condensed 700
2. Card grid: CSS grid, 1 column on mobile, 2 columns at `sm:` (640px), 3 columns at `lg:` (1024px)
3. Gap between cards: 24px (`gap-6`)
4. Each card: existing `Card` component with `href="/news/{slug}"`, optional `imageSrc`, children containing headline + DateDisplay

**Metadata:** `generateMetadata` returns title, description, openGraph, and twitter properties per established pattern.

**JSON-LD:** Breadcrumb JSON-LD via existing `breadcrumbJsonLd` + `JsonLdScript` utilities. Path: Home > News.

**Empty state:** Centered text block when `newsPosts.docs.length === 0`. Heading + body from copywriting contract above. No image or illustration.

### Existing components reused without modification:
- `Card` -- news post cards in grid
- `Section` -- page wrapper with max-width constraint
- `DateDisplay` -- relative date formatting on each card
- `JsonLdScript` -- breadcrumb structured data

### No component changes:
- `Button` -- not used on listing page
- `Logo` -- unchanged
- `Header` / `Footer` -- unchanged (Footer "News" link already points to `/news`)
