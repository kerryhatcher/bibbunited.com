  <action>
    **Gap 2 — News article 500 error:**

    In `src/app/(frontend)/news/[slug]/page.tsx`, line 87:
    Current: `(post.author as User).displayName || 'BIBB United Staff'`
    Fix to: `(post.author as User)?.displayName || 'BIBB United Staff'`

    The issue: `typeof null === 'object'` in JavaScript, so when `post.author` is `null`, the check `typeof post.author === 'object'` passes, then `(null as User).displayName` throws "Cannot read properties of null". Adding `?.` makes it safely return undefined, falling through to the `|| 'BIBB United Staff'` fallback.

    Note: Line 67 in `generateMetadata` already has `?.` — only line 87 in the render function is missing it.

    **Gap 3 — About page SEO fallback:**

    In `src/app/(frontend)/[slug]/page.tsx`, in `generateMetadata` (line 46-51):
    Current: `description: page.meta?.description || undefined`
    Fix to: `description: page.meta?.description || page.title + ' - BIBB United community information and resources'`

    This provides a meaningful fallback description when the CMS page has no SEO metadata configured. The About page (and any future CMS pages) will get a description derived from the page title rather than having no meta description at all, which is what causes the SEO score to drop below 95.
  </action>
  <verify>

  </verify>
  <acceptance_criteria>
    - news/[slug]/page.tsx line 87 contains `(post.author as User)?.displayName` (with optional chaining `?.`)

    - [slug]/page.tsx generateMetadata description uses fallback: `page.title + ' - BIBB United community information and resources'`
