**JsonLdScript component** -- XSS-safe JSON-LD renderer per Next.js official guidance. Uses `React.createElement('script', { type: 'application/ld+json' })` with innerHTML sanitized by replacing `<` with unicode `\u003c` escape to prevent script injection. NOTE: This is the documented safe pattern for JSON-LD -- the HTML sanitization concern does not apply because JSON-LD is data, not rendered HTML, and the escape prevents breaking out of the script tag.

**organizationJsonLd()** -- Returns `{ '@context': 'https://schema.org', '@type': 'Organization', name: 'BIBB United', url: 'https://www.bibbunited.com' }`.

**websiteJsonLd()** -- Returns `{ '@context': 'https://schema.org', '@type': 'WebSite', name: 'BIBB United', url: 'https://www.bibbunited.com' }`.

**newsArticleJsonLd(post)** -- Accepts `{ title: string, slug: string, publishDate: string, updatedAt: string, authorName: string, imageUrl?: string, description?: string }`. Returns NewsArticle schema with `headline`, `datePublished`, `dateModified`, `author` (Person), `publisher` (Organization), `mainEntityOfPage` (`${SITE_URL}/news/${post.slug}`). Conditionally includes `image` and `description` if provided.

**breadcrumbJsonLd(items)** -- Accepts `{ name: string, url: string }[]`. Returns BreadcrumbList with numbered ListItem entries.

**governmentOrgJsonLd(bodyName, officials)** -- Per D-06, accepts `bodyName: string` and `officials: { name: string, role: string, email?: string, phone?: string }[]`. Returns GovernmentOrganization with `member` array of Person schemas. Person includes `jobTitle` from role, `email` as `mailto:` URI, `telephone` from phone.
