# Quick Task: Remove gap between hero image and top menu

## Analysis

The `<header>` uses `sticky top-0` positioning, which means it occupies normal document flow space (unlike `fixed`). The `<main>` element has `pt-16` (padding-top: 4rem) which creates an unnecessary white gap between the header and the hero content. This padding would only be needed if the header used `position: fixed`.

## Task 1: Remove pt-16 from main element

- **files**: `src/app/(frontend)/layout.tsx`
- **action**: Remove `pt-16` class from `<main>` element
- **verify**: Visual check — hero image sits flush below header
- **done**: No white gap between header and hero image
